## Terms in the formal model

### identifier

An index to a source. An identifier for a source document is typically derived from the name of the document. An identifier for a source within a document is typically based on a numbering system that the document provides.

### applicability condition

The part of a requirement that says under what conditions the requirement applies. An applicability condition evaluates to true or false. When false, the requirement is not applicable, and an evaluator would report a result such as "not applicable" or "pass" for the requirement.

### expectation

Text that says what should, shall, or must occur for accessibility.

### rationale

A why-explanation.

### relationship

How items relate to each other.

### requirement

A source fragment that includes an expectation.

### source

A text that gives guidance on assessment of accessibility. It can be a whole document or any part of a document. Sources may contain sources.

### source document

A source, such as a standard or regulation, which is not part of a higher-level source.

### source fragment

Part of a source document.


## Terms not in the formal model

### clause

A source fragment within certain source documents. For example, in EN 301 549 a numbered clause may be normative or informative, and may contain other numbered clauses.

### provision

A source fragment within certain source documents. For example, EU directives contain provisions.

### success criterion

A requirement in certain source documents. For example, in WCAG 2, numbered success criteria are normative and do not contain other success criteria.
